PPE1: CCVEN PHP

Contexte

CCVEN est une association loi de 1901, qui joue le rôle de comité d'entreprise auprès des agents de l'Education Nationale. Pendant les congés scolaires, les villages peuvent être réservés par les agents adhérents et leur famille pour des séjours d'une ou plusieurs semaines. Ces séjours de vacances sont subventionnés en fonction du quotient familial. Les agents disposent sur place de services de restauration, d'animations, d'équipements sportifs... La réservation est alors ouverte à toute personne sur simple demande. Le CVVEN dispose d'un site vitrine, présentant les différents centres, et les tarifs des séjours disponibles. Jusqu'à présent les demandes de réservations sont gérées par téléphone, ou par un simple formulaire de demande de réservation en ligne. Aucun moyen n'est disponible, pour connaître en direct la disponibilité d'un équipement. Le conseil d'administration a décidé de faire appel à un prestataire informatique notamment pour développer le site web. Vous et votre équipe avez été retenus pour réaliser ce projet. Il est prévu de développer puis de déployer plusieurs applications:

  • Une application web permetent la réservation des séjours de vacances pour les agents (en utilisant le Framework PHP CodeIgniter et la librairie Bootstrap);
  • une application Java « client lourd » pour l'organisation des réunions (gestion des participants);
  • une application mobile destinée aux participants aux réunions permettant aux organisateurs de proposer des quizz ou des sondages en direct. Vous êtes engagés pour plusieurs mois dans le Jura au centre Paul Émile Victor (Les Rousses) qui dispose d'un important réseau informatique, et qui accueillera les différents serveurs. PPE 02: Conception de la base de données:

Durant cette mission nous avons été chargés de proposer une base de données pour une application de réservation des séjours de vacances en suivant les étapes suivantes:

  • 1. Construire le schéma conceptuel de données modélisant cette situation (faites des hypothèses si c’est nécessaire).
  • 2. Traduire le schéma conceptuel en schéma relationnel.
  • 3. Implémenter les différentes tables dans la base de données.



    Productions attendues

    • Schémas conceptuel et relationnel proposés
    • Script SQL de création des tables ainsi que son jeu d’essai.

    PPE 03: Réservation des séjours

    Nous sommes chargés de développer un module de réservation de séjours pour les utilisateurs. Celui-ci doit disposer des fonctionnalités suivantes:

    • Formulaire de réservation des séjours. Une réservation dure toujours une semaine dusamedi au samedi. Attention: les semaines hors vacances scolaires sont de toute façonindisponibles pour les séjours de vacances
    • Possibilité de choisir la «pension complète», ou la «demi-pension», etoptionnellement un forfait «ménage fin de séjour».
    • Affichage des réservations d'un utilisateur en particulier ou de tous les utilisateurs(éventuellement dans un calendrier → voir PPE05 optionnel).
    • Possibilité pour un utilisateur d'annuler une demande de réservation tant qu'elle n'a pas été validée par un administrateur du site
    • Contrôler la validité des données saisies dans le formulaire.

    PPE 04: Authentification des utilisateurs

    La mission du PPE 04 consite à réaliser un module d'authentification des utilisateurs du site de réservation. Celui-ci disposera des fonctionnalités suivantes :

    • formulaire de création de compte utilisateur
    • possibilité de changer son mot de passe via un formulaire
    • authentification des utilisateurs enregistrés dans la base de données (mise en place d'une session)
    • bouton de déconnexion
    • assurer une navigation fluide entre les différentes pages de l’application
